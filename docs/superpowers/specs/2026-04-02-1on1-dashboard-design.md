# 1on1 Dashboard — Design Spec
_Date: 2026-04-02_

## Overview

A stakeholder dashboard for the 1on1 product's registration/payment data. Data lives in a Google Sheet ("mixpanel dashboard"), updated daily by the team. A Python cron script pulls the sheet, generates `data.json`, and pushes it to a GitHub repo. GitHub Pages serves the static HTML dashboard.

---

## Architecture

```
/
├── index.html        ← static dashboard (GitHub Pages)
├── data.json         ← generated by fetch_data.py, committed to repo
└── fetch_data.py     ← run via cron on local machine
```

`gsheet_cred.json` is stored only on the machine running the cron job — never committed to the repo.

**Flow:**
1. Cron runs `fetch_data.py` (daily or on-demand)
2. Script reads Google Sheet via pygsheets using service account
3. Processes and aggregates data → writes `data.json`
4. Script does `git add data.json && git commit -m "Update dashboard data [date]" && git push`
5. GitHub Pages auto-serves updated `data.json` and `index.html`

## Data Processing

### Cleaning
- Strip `₹` and `,` from revenue/ARPU columns, parse as float
- Parse dates — handle both `13-May-25` and `13/05/2025` formats
- Skip rows with empty dates

### Aggregation

**Summable fields (SUM across period):**
- `total_payments`, `new_payments`, `repeat_payments`
- `total_revenue`, `new_revenue`, `repeat_revenue`
- Platform: `web`, `android`, `ios` counts and revenues

**Recalculated fields (never summed directly):**
- `arpu` = `total_revenue / total_payments` (0 if no payments)
- `new_arpu` = `new_revenue / new_payments` (0 if no new payments)
- `repeat_arpu` = `repeat_revenue / repeat_payments` (0 if no repeat)
- `new_pct` = `new_payments / total_payments * 100`
- `repeat_pct` = `repeat_payments / total_payments * 100`
- Platform %: `web_pct = web / total_payments * 100`, same for iOS/Android
- Platform ARPUs: `web_arpu = web_revenue / web` (0 if 0 payments)

**Weekly:** group by ISO week (Monday start)  
**Monthly:** group by calendar month (YYYY-MM)

### Output: data.json

```json
{
  "last_updated": "2026-04-02",
  "overall":  { "daily": [...], "weekly": [...], "monthly": [...] },
  "web":      { "daily": [...], "weekly": [...], "monthly": [...] },
  "ios":      { "daily": [...], "weekly": [...], "monthly": [...] },
  "android":  { "daily": [...], "weekly": [...], "monthly": [...] },
  "platform": { "daily": [...], "weekly": [...], "monthly": [...] }
}
```

## Python Script (`fetch_data.py`)

Dependencies: `pygsheets`, `pandas`

Steps:
1. Authorize with pygsheets using service account creds file
2. Open spreadsheet by key
3. Read each of the 5 sheets into a DataFrame
4. Clean currency strings and parse dates
5. Build daily data (rows as-is after cleaning)
6. Aggregate to weekly and monthly (sum + recalculate)
7. Build `data.json` dict with all tabs and granularities
8. Write `data.json` to repo root
9. `git add data.json && git commit -m "Update dashboard data YYYY-MM-DD" && git push`

The script takes the repo path as a constant at the top so it's easy to configure.
